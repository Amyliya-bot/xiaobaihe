# 参与开发

感谢参与小白盒。修改前请先阅读 `README.md`、`项目介绍.md`、`PRIVACY.md` 和当前修改相关的源码与测试。

## 基本要求

1. 核心流程保持离线，不增加账号、遥测、广告或后台上传。
2. 优先使用普通用户看得懂的交互和文字，不把专业建模参数直接推到主流程。
3. 不覆盖或静默降低用户原始模型质量；危险操作必须明确并可撤回。
4. 新依赖需要记录用途、锁定版本、维护状态和许可证。
5. 新功能必须实际可操作，并补与风险相称的单元或 Electron 端到端测试。

## 提交前验证

```powershell
cd app
npm install
npm run verify
npm run test:e2e
```

涉及三维画布时，还应在 1440×900 和 1280×720 下检查非空画面、相机取景、文字溢出和控件遮挡。不要提交 `app/release`、`app/test-results`、本地工程或用户素材。
