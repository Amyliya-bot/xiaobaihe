# 小白盒｜项目介绍

## 一句话介绍

这是一个面向普通创作者的开源 Windows 桌面工具，帮助没有三维建模经验的用户快速搭建白膜场景、设计镜头和人物动作，并将结果导出为可用于 AI 图像/视频平台及常用三维软件的参考素材。

## 项目简介

在 AI 图像和视频创作中，仅靠文字往往难以准确描述空间布局、镜头角度、人物站位和运动过程；而 Blender、Maya 等专业三维软件的学习成本又比较高。本项目希望填补两者之间的空白：用拖拽、简单变形、人物摆姿势和简化时间轴，让普通用户也能快速把脑中的空间与动作表达出来。

用户可以在本地搭建场景、摆放基础物体或导入现有模型，设置相机和画幅，保存多个镜头，并为相机、物体和人台记录动画状态。完成后，软件可根据目标用途导出图片、视频、三维模型，以及深度图、边缘图、法线图、对象色块图和蒙版等 AI 控制素材。

项目第一阶段不接入 AI，也不负责直接生成图片或视频。它更像是 AI 创作流程前端的“空间与镜头草稿工具”：先帮助用户把构图、空间、运镜和动作确定下来，再把标准化素材交给不同的生成平台继续创作。

## 核心特点

- **面向零基础用户**：用直观的拖拽、控制柄和普通语言替代复杂的专业建模操作。
- **场景、镜头与动作一体化**：支持基础建模、模型导入、人物摆姿、相机镜头和简化动画时间轴。
- **按用途校验导出**：视频预设会检查已核对的格式、尺寸、比例、帧率与时长，不静默修改用户内容。
- **丰富的 AI 控制素材**：可从同一三维场景生成像素级对齐的主图、深度图、边缘图、法线图、对象色块图和蒙版。
- **离线与本地优先**：核心功能完全离线运行，工程保存在用户电脑中，不上传模型、图片、视频或使用数据。
- **开放和可扩展**：源码按 MIT License 准备公开，平台适配规则采用独立配置，便于随 AI 平台变化持续更新。

## 典型使用场景

- 为 AI 图片生成准备更稳定的构图、透视和空间参考。
- 为 AI 视频生成设计首尾帧、运镜、物体运动和人物动作。
- 快速制作分镜、短视频预演、课堂演示和概念动画。
- 搭建室内、建筑、产品或游戏概念场景。
- 将轻量场景导出到其他三维软件中继续编辑。

## 基本工作流程

1. 添加方块、圆柱、球体、墙体、地面和简化人台，或导入 GLB、GLTF、OBJ 模型。
2. 调整物体的位置、旋转、大小、形状和颜色，完成人物站位与姿势设计。
3. 设置相机、画幅和多个镜头，选择平滑运镜或直接切换。
4. 在简化时间轴中记录相机、物体和人物的关键状态并预览动画。
5. 选择目标用途或 AI 平台，由软件校验并导出对应的图片、视频、模型或控制素材包。

## 当前可用版本

当前 Windows 公开 Beta 支持 GLB、GLTF、OBJ 模型导入与导出、PNG 图片与六通道控制图、逐帧 PNG，以及 MP4/H.264 动画视频。技术方案采用 Electron、React、TypeScript、Three.js、Chromium WebCodecs 与 Mediabunny，并通过本地 `.block3d` 工程文件支持保存、自动恢复和继续编辑。

首批适配方向覆盖普通图片参考、构图参考、ControlNet 类专用控制、视频首尾帧、参考视频、视频重绘和人物动作迁移等工作流。平台规则会记录核对日期；对于缺少公开稳定标准的平台，将明确提示需要实机复核，不宣称“一种格式适配所有平台”。

## 项目边界

本项目不是缩小版 Blender，也不是 AI 生成平台。第一版不包含高级雕刻、复杂拓扑、影视级材质与渲染、专业视频剪辑、云端协作和账号体系。项目关注的是一件事：让普通人用尽可能低的学习成本，快速、清楚地表达空间、镜头和动作。

## 当前状态

项目已进入 Windows 公开 Beta 阶段，并会持续更新。当前已提供 GitHub 开源仓库、MIT License、Windows 安装包、SHA-256 校验值和运行时第三方许可说明；仍建议重要工程保留备份，并欢迎通过 GitHub Issues 反馈问题、兼容性结果和改进建议。

### 可直接引用的短版介绍

小白盒是一款面向普通创作者的开源 Windows 桌面工具。它通过简单的拖拽建模、人物摆姿、相机镜头和简化时间轴，帮助没有三维基础的用户快速设计空间、构图、运镜与动作，并按不同 AI 图像/视频平台的要求导出参考图、首尾帧、参考视频、三维模型及深度图、边缘图、法线图等控制素材。项目核心功能完全离线，不接入 AI、不产生推理费用，也不上传用户数据。
